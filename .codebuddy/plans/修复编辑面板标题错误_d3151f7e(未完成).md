## 用户需求

修复点击任务打开编辑面板时，面板标题显示"新建任务"而不是"编辑任务"的问题。

## 问题分析

1. 点击任务时，TaskSlider 调用 openEditPanel，正确设置 editingTask
2. 点击事件冒泡到 DayView 容器（@click="handleTimeAxisClick"）
3. handleTimeAxisClick 调用 openCreatePanel，重置 editingTask 为 null
4. 导致面板标题计算为"新建任务"

## 核心功能

- 在 TaskSlider 的 handlePointerDown 中阻止事件冒泡，防止触发父容器的点击处理函数

## 修改文件

`src/components/task/TaskSlider.vue`

## 修改内容

在 `handlePointerDown` 函数开头添加 `event.stopPropagation()`：

```typescript
function handlePointerDown(event: PointerEvent, type: 'move' | 'resize-start' | 'resize-end') {
  // 阻止事件冒泡，避免触发父容器的点击事件
  event.stopPropagation()
  
  if (!sliderRef.value) return
  // ... 其余代码不变
}
```

## 实现原理

- `stopPropagation()` 阻止事件继续向上冒泡
- DayView 容器的 `@click="handleTimeAxisClick"` 将不会被触发
- editingTask 保持为正确的任务对象，面板标题正确显示为"编辑任务"
