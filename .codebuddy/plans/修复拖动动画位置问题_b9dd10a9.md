## 用户需求

修复从任务列表拖动任务时的两个问题：

1. **电脑端**：拖动动画位置不对，不会跟随鼠标移动
2. **手机端**：无法成功拖动，拖动动画定格在屏幕中间

## 核心问题

- 拖动预览位置不更新或不正确
- 移动端 HTML5 draggable API 支持有限，需要 touch 事件

## 问题根因

### 电脑端位置不跟随

1. `startDragFromList` 没有设置 `originalPosition`，初始值为 {x:0, y:0}
2. `document.addEventListener('drag', ...)` 事件触发不可靠
3. 需要改用更可靠的位置更新机制

### 手机端无法拖动

1. HTML5 `draggable` API 在移动端支持有限
2. 需要添加 touch 事件支持

## 技术方案

### 1. 修改 drag.ts - startDragFromList

- 接收初始位置参数 `position: DragPosition`
- 正确设置 `originalPosition` 和 `currentPosition`

### 2. 修改 TaskListDrawer.vue

- `handleDragStart` 传递初始位置给 `startDragFromList`
- 改用 `document.addEventListener('dragover')` 监听位置（比 drag 事件更可靠）
- 添加 touch 事件支持：
- `touchstart` + 长按检测触发拖拽
- `touchmove` 更新位置
- `touchend` 结束拖拽并放置

### 3. 触摸事件处理逻辑

```
touchstart -> 500ms 长按 -> 开始拖拽（设置 isDragging, 显示 DragPreview）
touchmove -> 更新 currentPosition
touchend -> 计算目标位置，调用 drop 处理
```
