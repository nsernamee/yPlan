## Product Overview

修复日历上 TaskSlider（已有日程滑块）拖动时不平滑的问题，使其与从任务列表拖动的平滑效果一致。

## Core Features

- 将时间粒度 `TIME_GRANULARITY` 从 15 分钟改为 **1 分钟**
- 拖动任务时精确到每分钟，不再跳格到 15/30/45 整点
- 桌面端和移动端的 TaskSlider 拖动均生效
- 影响范围仅限于 `offsetToMinutes` 和 `pixelsToTime` 两个函数

## Tech Stack

- Vue 3 + TypeScript
- Pinia 状态管理

## Implementation Approach

**核心改动**：将 `TIME_GRANULARITY` 常量值从 `15` 改为 `1`。这是最简单且影响最小的修改方案。

**为什么可行：**

- `Math.round(totalMinutes / 1) * 1 = Math.round(totalMinutes)`，即四舍五入到最近整数分钟
- 所有依赖此常量的代码自动生效，无需额外改动
- 不改变任何函数签名或调用方式
- 不影响其他功能逻辑

**影响范围分析：**

- `offsetToMinutes()` — 返回精确的整数分钟偏移量（而非 15 的倍数）
- `pixelsToTime()` — 返回精确到分钟的 HH:mm 时间字符串
- 受影响组件：TaskSlider（拖动/缩放）、DayView、WeekView 的所有时间计算

**性能考量：**

- 无性能开销变化，只是除法常量从 15 变为 1

## Architecture Design

单点修改，无需架构变更。

```
constants.ts (TIME_GRANULARITY: 15 → 1)
    └── time.ts (offsetToMinutes, pixelsToTime 自动生效)
        ├── TaskSlider.vue (onMove2D/onEnd2D 回调)
        ├── DayView.vue (handleDragOver/handleDrop)
        └── WeekView.vue (handleDragOver/handleDrop)
```
