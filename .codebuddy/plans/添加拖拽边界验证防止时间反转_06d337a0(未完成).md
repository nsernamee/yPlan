## 用户需求

在拖拽调整任务时间时添加边界判断，防止出现时间反转的情况（开始时间 > 结束时间）。

## 核心功能

1. **防止时间反转**：调整开始时间时不能超过结束时间，调整结束时间时不能早于开始时间
2. **最小持续时间限制**：任务持续时间不能少于 15 分钟
3. **实时反馈**：拖拽过程中限制拖拽范围，而不是只在结束时修正

## 问题分析

### 当前问题位置

**文件**: `src/components/task/TaskSlider.vue`

1. **resize-start**（第 136-141 行）：没有验证新的开始时间是否超过结束时间
2. **resize-end**（第 142-147 行）：没有验证新的结束时间是否早于开始时间
3. **onMove2D**（第 86-91 行）：只有最小高度限制（30px），没有时间逻辑验证

### 预期行为

- resize-start：向上拖拽时，新开始时间 ≤ 结束时间 - 15分钟
- resize-end：向下拖拽时，新结束时间 ≥ 开始时间 + 15分钟
- 任务持续时间始终 ≥ 15分钟

## 技术栈

- Vue 3 + TypeScript
- 现有工具函数：`calculateDuration`, `offsetTime`, `offsetToMinutes`

## 实现方案

### 1. 添加最小持续时间常量

**文件**: `src/utils/constants.ts`

添加 `MIN_DURATION = 15` 常量，表示任务最小持续时间（分钟）

### 2. 修改 TaskSlider.vue

#### onMove2D 函数修改

在拖拽过程中限制拖拽范围：

- **resize-start**: 计算最大允许的向上拖拽距离（当前持续时间 - 最小持续时间），限制 offset.y 不超过此值
- **resize-end**: 计算最大允许的向下拖拽距离，限制 offset.y 不超过此值

#### onEnd2D 函数修改

在结束时进行最终验证：

- **resize-start**: 验证新开始时间 < 结束时间 - MIN_DURATION
- **resize-end**: 验证新结束时间 > 开始时间 + MIN_DURATION
- 如果违反边界，自动修正到边界值

## 边界计算逻辑

```
resize-start（向上拖拽减小开始时间）:
  当前持续时间 = endTime - startTime
  最大向上拖拽分钟数 = 当前持续时间 - MIN_DURATION
  最大向上拖拽像素 = 最大向上拖拽分钟数 * (HOUR_HEIGHT / 60)
  
resize-end（向下拖拽增加结束时间）:
  当前持续时间 = endTime - startTime  
  最大向下拖拽分钟数 = 当前持续时间 - MIN_DURATION
  最大向下拖拽像素 = -(最大向下拖拽分钟数 * (HOUR_HEIGHT / 60))
```
