## 用户需求

修改桌面端和移动端的拖动逻辑，实现平滑拖动：

- 虚线指示器显示的时间就是实际创建的日程时间
- 不再四舍五入到整点，精确到分钟
- 默认日程时长保持 1 小时（如拖到 14:35，创建 14:35-15:35）

## 核心功能

1. DayView 的 handleDrop 和 handleCustomDrop 使用精确时间
2. WeekView 的 handleDrop 和 handleCustomDrop 使用精确时间
3. 使用 dragStore.targetTime 替代 Math.floor 整点计算

## 技术方案

### 问题根因

当前 `handleDrop` 和 `handleCustomDrop` 使用 `Math.floor(y / HOUR_HEIGHT)` 计算整点小时：

```typescript
const hour = Math.floor(y / HOUR_HEIGHT)
const startTime = `${hour}:00`  // 整点
```

### 解决方案

直接使用 `dragStore.targetTime`（已在 handleDragOver/handleTouchMove 中精确计算到分钟）：

```typescript
const startTime = dragStore.targetTime || `${hour}:00`
// 计算 endTime = startTime + 1小时
```

### 修改文件

**1. src/components/calendar/DayView.vue**

- `handleDrop`: 使用 targetTime，计算精确的 endTime
- `handleCustomDrop`: 同样使用 targetTime

**2. src/components/calendar/WeekView.vue**

- `handleDrop`: 使用 targetTime
- `handleCustomDrop`: 使用 targetTime

### 时间计算逻辑

```typescript
// 获取精确开始时间
const startTime = dragStore.targetTime || '00:00'

// 计算1小时后的结束时间
const [h, m] = startTime.split(':').map(Number)
const endHour = h + 1
const endTime = `${endHour.toString().padStart(2, '0')}:${m.toString().padStart(2, '0')}`
```
